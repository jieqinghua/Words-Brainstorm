# 创意关键词脑暴

当用户输入一个主题词时，执行以下流程。

## 工作流

### 第一步：选择创意方法

从 [references/creativity-methods.md](references/creativity-methods.md) 中随机选取一种方法。每次执行时优先选择与上次不同的方法，以保持多样性。

### 第二步：生成发散词

使用选定的方法，围绕用户输入的主题词生成 5 个发散词。发散词应该：
- 与主题词来自不同领域或语义空间，产生意外感
- 不能是主题词的同义词或常见搭配词
- 词性不限（名词、动词、形容词皆可）

### 第三步：寻找关联点

对每组 主题词↔发散词，从多个维度寻找两者的自然关联。关联可以来自：

- **感官特征**：颜色、形状、声音、气味、触感（如"火"的红色 → "珊瑚"的红色 → "红珊瑚"）
- **功能用途**：做什么、怎么用、在什么场景出现（如"火"用来烧烤 → "花甲"可以烤 → "花甲烧烤"）
- **文化典故**：成语、俗语、歌词、电影、品牌名中的既有联系
- **生活场景**：日常中两个词同时出现的画面（厨房、户外、节日等）
- **情感共鸣**：两个词唤起的相似情绪或感受

**关键**：不要停留在词的表面拼凑，要深入到一个具体的、可感知的连接点。好的关联能让人立刻在脑海中看到画面。

### 第四步：创意组合

基于找到的关联点，产出一个创意词或短句。核心要求：

1. **自然不牵强**：组合结果应该是生活中可能真实出现的说法，而不是硬造新词，意料之外又情理之中
   - ✅ "红珊瑚"（火的颜色 + 珊瑚的形态，自然成立）
   - ✅ "花甲烧烤"（火的功能 + 花甲的吃法，生活场景）
   - ❌ "火珊瑚"（强行拼接，没有真实感）
   - ❌ "火贝"（机械组合，不知所云）

2. **关联路径越长越有趣**：好的创意有多层推导逻辑，不是一步到位
   - "火 → 红色 → 珊瑚的颜色 → 红珊瑚"（3步，有推导过程）
   - 而非 "火 + 珊瑚 = 火珊瑚"（1步，简单拼接）

3. **优先用已有词汇**：组合结果可以是现存词汇，创新在于你发现了它和主题词之间的关联

## 输出格式

对每组结果输出三行：

```
🎯 发散词：[发散词]
🔗 组合结果：[创意词/短句]
💡 关联逻辑：[一句话解释为什么这样组合]
```

5 组全部输出后，末尾加一行简短总结。

## 关键原则

- **自然优于新奇**：宁可用一个真实的词汇发现巧妙的关联，也不要生造一个看起来很"创意"但没有生命力的拼接词
- **多层推导**：好的关联路径至少走2-3步（火→红色→珊瑚→红珊瑚），一步到位的拼接（火+珊瑚=火珊瑚）太生硬
- **画面感检验**：如果读者听完解释后能在脑海中清晰看到一个画面，就是好创意。看不到画面的，就是牵强
- **拒绝套路**：避免常见组合，每次都要不一样的发散方向
